Подход с **code-bridge.nvim** — это, пожалуй, самый элегантный способ совместить лучшее из двух миров: мощь терминального AI-агента (вроде Claude Code или OpenCode) и привычный интерфейс Neovim[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim).

Главная идея здесь не в том, чтобы заменить агента, а в том, чтобы построить удобный «мост» между редактором и агентом, работающим в соседнем окне терминала[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim).

---

### 🧠 Как это работает изнутри

Плагин не запускает AI сам. Он отправляет контекст (код, диагностику, diff) из Neovim в уже запущенный процесс AI-агента. Делает он это одним из двух способов[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim):

1. **Через tmux** (рекомендуемый): Находит нужную панель (например, с запущенным `claude` или `opencode`) и «вводит» туда ваш запрос, как если бы вы печатали его сами. При этом он может сразу переключить вас в ту панель, чтобы видеть ответ.
    
2. **Через буфер обмена**: Если tmux недоступен, просто копирует контекст в буфер обмена — вы сами вставляете его в терминал.
    

Для максимальной эффективности вы создаёте tmux-сессию с двумя окнами/панелями: в одной редактируете код (Neovim), в другой — живет AI-агент. Плагин служит связующим звеном между ними.

### 🎮 Команды, которые меняют всё

Вместо того чтобы копировать код мышкой, вы используете команды Neovim. Вот ключевые из них[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim):

#### 📤 Отправка контекста в агента

|Команда|Что делает|
|---|---|
|`:CodeBridgeTmux`|Отправляет текущий файл (или выделенный фрагмент в визуальном режиме) агенту.|
|`:CodeBridgeTmuxAll`|Отправляет **все открытые буферы** — идеально, когда проблема затрагивает несколько файлов.|
|`:CodeBridgeTmuxDiff`|Отправляет **незакоммиченные изменения** (git diff). Отлично подходит для ревью или авто-написания commit message.|
|`:CodeBridgeTmuxDiagnostics`|Отправляет ошибки и предупреждения от LSP для текущего файла. Лучший способ сказать: «Посмотри на эти ошибки и исправь их».|

#### 💬 Интерактивное общение

- **`:CodeBridgeTmuxInteractive`**: Открывает всплывающее окно в Neovim, где вы можете написать или отредактировать сложный промпт, прежде чем отправить его агенту. Отправка по `Ctrl-s`[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim).
    
- **`:CodeBridgeQuery`**: Это **встроенный чат**! Открывает буфер с markdown-разметкой прямо в Neovim. Вы пишете запрос, он уходит агенту, и ответ возвращается в этот буфер. Сессия сохраняется, пока вы не закроете окно[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim).
    

#### 🎨 Продвинутая настройка

Плагин предлагает три режима поиска вашей tmux-панели с агентом, что даёт полный контроль над рабочим процессом[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim):

- **`window_name` (по умолчанию)**: Ищет окно tmux с именем `claude`. Проще всего.
    
- **`current_window`**: Ищет процесс в текущем окне — подходит, если вы сами переключаетесь между панелями.
    
- **`find_process`**: Честно сканирует все панели в поисках процесса. Самый гибкий, но чуть более медленный вариант.

### ⚡ Почему это круто на практике

1. **Контекст не теряется**: Вам не нужно копировать ошибку из LSP в терминал — `:CodeBridgeTmuxDiagnostics` делает это за вас.
    
2. **Контроль остаётся за вами**: Вы по-прежнему видите каждый ответ агента в терминале и принимаете решение о применении изменений. Плагин ничего не делает за вашей спиной.
    
3. **Стабильность**: Ваша tmux-сессия с AI-агентом переживёт перезапуск Neovim или даже внезапное закрытие ноутбука (если вы на сервере).
    

### 🔧 Быстрый старт

Установите плагин через `lazy.nvim`[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim):
```lua
{
  "samir-roy/code-bridge.nvim",
  config = function()
    require('code-bridge').setup()
  end
}
```

А затем добавьте удобные хоткеи (пример)[](https://explore.market.dev/ecosystems/neovim/projects/code-bridge-nvim):
```lua
-- Отправить файл/выделенный код
vim.keymap.set("n", "<leader>ct", ":CodeBridgeTmux<CR>")
vim.keymap.set("v", "<leader>ct", ":CodeBridgeTmux<CR>")
-- Отправить все буферы
vim.keymap.set("n", "<leader>ca", ":CodeBridgeTmuxAll<CR>")
-- Отправить git diff
vim.keymap.set("n", "<leader>cd", ":CodeBridgeTmuxDiff<CR>")
-- Открыть чат внутри Neovim
vim.keymap.set("n", "<leader>cq", ":CodeBridgeQuery<CR>")
```
`code-bridge.nvim` — это пример прагматичного подхода: он не пытается заменить ни Neovim, ни AI-агента, а делает их работу в связке максимально комфортной для вас.
